### The Conceptual Breakdown: How Quicksort Works

Quicksort (Quick Sort) is a **Divide and Conquer** algorithm. It works by recursively breaking a large problem into smaller, manageable sub-problems until the entire list is sorted.

Here is the step-by-step logic:

#### Step 1: Choose a "Pivot"
The algorithm selects one element from the array to act as a reference point. This element is called the **Pivot**.
*   *Common choices:* The first element, the last element, or a random element.
*   *Goal:* The pivot determines where the rest of the elements will go.

#### Step 2: Partitioning (The Split)
The algorithm iterates through the rest of the array and compares every element to the Pivot.
*   **Elements smaller than the Pivot** are moved to the **Left** side.
*   **Elements larger than the Pivot** are moved to the **Right** side.
*   *Note:* Elements equal to the pivot can go to either side (usually handled with the pivot itself).

At this stage, the Pivot is in its "correct" final position relative to the other elements. The Left side contains smaller numbers, and the Right side contains larger numbers.

#### Step 3: Recursive Calls
The algorithm now applies the exact same logic (Steps 1 and 2) to the **Left** sub-array and the **Right** sub-array.
*   It picks a new pivot for the Left side.
*   It picks a new pivot for the Right side.
*   It partitions them again.

#### Step 4: The Base Case (Stop Condition)
The recursion continues until the sub-arrays become very small (size 1 or 0).
*   An array of size 1 is already sorted.
*   An array of size 0 is empty.
*   Once the recursion hits this base case, it stops, and the sorted sub-arrays are stitched back together.

### Concrete Example Walkthrough

Let's sort the list: `[3, 6, 12, 5, 9]`

**Iteration 1:**
*   **Pivot:** We pick the first element, `3`.
*   **Partition:**
    *   `6, 12, 5, 9` are all larger than `3`. They go to the **Right**.
    *   There are no elements smaller than `3`. The **Left** is empty.
*   **Result:** `[Left: []] + [Pivot: 3] + [Right: [6, 12, 5, 9]]`

**Iteration 2 (Recursive call on Right):**
*   **Array:** `[6, 12, 5, 9]`
*   **Pivot:** We pick the first element, `6`.
*   **Partition:**
    *   `5` is smaller than `6`. It goes to **Left**.
    *   `12, 9` are larger than `6`. They go to **Right**.
*   **Result:** `[Left: [5]] + [Pivot: 6] + [Right: [12, 9]]`

**Iteration 3 (Recursive calls on Left and Right):**
*   **Left Sub-array:** `[5]` (Base case reached, it is sorted).
*   **Right Sub-array:** `[12, 9]`
    *   **Pivot:** `12`.
    *   **Partition:** `9` is smaller than `12`.
    *   **Result:** `[Left: [9]] + [Pivot: 12] + [Right: []]`

**Final Stitching:**
We combine the sorted pieces: `[5]` + `[6]` + `[9]` + `[12]` = `[5, 6, 9, 12]`. (Wait, we missed the original pivot `3`! The full list is `[3, 5, 6, 9, 12]`).

### Python Implementation

Here is a Python implementation of Quicksort. This version uses **List Comprehensions** to make the code readable and concise, though it creates new lists (taking extra memory).

```python
def quicksort(arr):
    # Base Case: If the array has 1 or 0 elements, it is already sorted.
    if len(arr) < 2:
        return arr
    
    # Step 1: Choose a Pivot
    # We pick the last element as the pivot (common strategy).
    pivot = arr[-1]
    
    # Step 2: Partitioning
    # We split the array into three lists:
    # 1. Elements smaller than the pivot
    # 2. The pivot itself
    # 3. Elements larger than (or equal to) the pivot
    
    left = [x for x in arr if x < pivot]
    right = [x for x in arr if x > pivot]
    
    # Step 3: Recursive Calls
    # We apply quicksort to the left and right lists, then stitch them together.
    return quicksort(left) + [pivot] + quicksort(right)

# --- Example Usage ---
data = [3, 6, 12, 5, 9]
sorted_data = quicksort(data)
print(f"Original: {data}")
print(f"Sorted:   {sorted_data}")
```

### Complexity Analysis

*   **Time Complexity (Average):** $O(n \log n)$. This is the reason Quicksort is so popular—it is very fast for large datasets.
*   **Time Complexity (Worst):** $O(n^2)$. This happens if the Pivot is always the largest or smallest element (e.e., the array is already sorted).
*   **Space Complexity:** $O(n)$. The recursive calls require memory on the "stack" to store the sub-arrays.
