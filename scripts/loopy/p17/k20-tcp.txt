        *   `ssthresh` (Slow Start Threshold): The boundary between Slow Start and Congestion Avoidance.

    *   *The States:*

        2.  **Congestion Avoidance:** Linear growth.
        3.  **Event:** Packet Loss (Timeout or Triple Duplicate ACK).
    *   *The Recovery Mechanisms:*
        1.  **Fast Retransmit:** Detecting loss via duplicate ACKs.
        2.  **Fast Recovery:** Handling the loss without resetting the window to zero.

3.  **Drafting the Content - Section by Section:**

    *   **Introduction:** Define the goal (prevent network collapse) and the variables (`cwnd`, `ssthresh`).

    *   **Phase 1: Slow Start:**
        *   *Concept:* Start small, grow fast.
        *   *Mechanism:* `cwnd` starts at 1 MSS (Maximum Segment Size).
        *   *Growth:* Every ACK received increases `cwnd` by 1 MSS. (Exponential growth: 1, 2, 4, 8...).
        *   *Termination:* When `cwnd` >= `ssthresh`.

    *   **Phase 2: Congestion Avoidance:**
        *   *Concept:* Once the network capacity is roughly known, grow slower to avoid hitting the limit.
        *   *Mechanism:* Add 1 MSS to `cwnd` per RTT (Linear growth).
        *   *Why "Avoidance"?* It's slower, so it's less likely to cause congestion.

    *   **Phase 3: Handling Packet Loss (The Trigger):**
        *   *Scenario A: Timeout.* (Old school, less relevant now but still exists). `cwnd` = 1 MSS, `ssthresh` = `cwnd` / 2.
        *   *Scenario B: Triple Duplicate ACKs.* (The modern standard). This triggers Fast Retransmit.

    *   **Phase 4: Fast Retransmit:**
        *   *Concept:* Don't wait for the timer. If I get 3 ACKs for the same sequence number, I know the next packet was dropped.
        *   *Action:* Re-transmit the missing packet immediately.

    *   **Phase 5: Fast Recovery:**
        *   *Concept:* Don't go back to Slow Start (which is slow). Keep the window high but adjust it.
        *   *Mechanism:*
            1.  Set `ssthresh` = `cwnd` / 2.
            2.  Set `cwnd` = `ssthresh` + 3 MSS (or 3 segments).
            3.  Send the retransmitted packet.
            4.  For every *new* ACK received, increase `cwnd` by 1 MSS (to "push" the window forward).
            5.  When the missing packet is acknowledged, enter Congestion Avoidance (standard linear growth).
