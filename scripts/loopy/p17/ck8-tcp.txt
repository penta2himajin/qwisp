5.  **Drafting Content - Section by Section:**

    *   *Introduction:* TCP is a reliable protocol. If the network is congested, packets drop. TCP needs to detect this and slow down. It uses packet loss as a signal.

    *   *Variables:*
        *   `cwnd` (Congestion Window): Sender's limit.
        *   `ssthresh` (Slow Start Threshold): The point where Slow Start switches to Congestion Avoidance.

    *   *Slow Start:*
        *   *Mechanism:* Start with 1 MSS (Maximum Segment Size).
        *   *Growth:* Every ACK received doubles the window (Exponential).
        *   *Goal:* Find the bandwidth quickly.
        *   *Transition:* When `cwnd` hits `ssthresh`, switch to Congestion Avoidance.

    *   *Congestion Avoidance:*
        *   *Mechanism:* Add 1 MSS per RTT (Round Trip Time).
        *   *Growth:* Linear (Additive Increase).
        *   *Goal:* Probe for more bandwidth without causing congestion.

    *   *Handling Loss (The "Three-legged" approach):*
        *   *Scenario A: Timeout (3-way handshake for recovery).*
            *   If a timer expires, assume severe congestion.
            *   Reset `cwnd` to 1 MSS.
            *   Reset `ssthresh` to half the current window.
            *   Restart Slow Start.
        *   *Scenario B: Fast Retransmit (3x Dup ACKs).*
            *   Receiver sends 3 duplicate ACKs (indicating a gap).
            *   Sender retransmits the missing packet immediately (don't wait for timer).
            *   *Crucial Detail:* This implies *some* bandwidth is still available (not total collapse).
        *   *Fast Recovery:*
            *   *Mechanism:* Don't go back to Slow Start.
            *   *Action:* Set `ssthresh` to half the window.
            *   *Action:* Set `cwnd` to `ssthresh` + 3 (or 6, depending on the specific algorithm like F-RACK, but standard is usually `ssthresh + 3 MSS`).
            *   *Why?* We know 3 packets got through, so the pipe isn't empty. We can keep sending.
